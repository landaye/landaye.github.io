React

文章内容

. 说说对 React 的理解？有哪些特性？

  1. 用于构建用户界面的 JavaScript 库,虚拟 DOM 来有效地操作 DOM，遵循从高阶组件到低阶组件的单向数据流.

  2. 特性

    1. JSX 语法
    2. 单向数据绑定
    3. 虚拟 DOM
    4. 声明式编程
    5. Component

. 说说 Real DOM 和 Virtual DOM 的区别？优缺点？

  1. Virtual Dom，本质上是以 JavaScript 对象形式存在的对 DOM 的描述
  2. 区别:
    虚拟 DOM 不会进行排版与重绘操作,而真实 DOM 会频繁重排与重绘

. 说说 React 生命周期有哪些不同阶段？每个阶段对应的方法是？

  1. 挂载阶段（Mounting）：
    1. constructor(props)：组件的构造函数，用于初始化 state 和绑定事件处理方法。
    2. static getDerivedStateFromProps(props, state)：用于根据新的 props 更新 state。
    3. render()：返回要渲染的 React 元素，是必须实现的方法。
    4. componentDidMount()：在组件挂载后立即调用，适合进行网络请求、订阅事件等初始化操作。
  2. 更新阶段（Updating）：
    1. static getDerivedStateFromProps(props, state)：在此阶段也会被调用。
    2. shouldComponentUpdate(nextProps, nextState)：用于控制组件是否重新渲染。
    3. render()：重新渲染组件。
    4. getSnapshotBeforeUpdate(prevProps, prevState)：在最近一次渲染输出（提交到 DOM 节点）之前调用。
    5. componentDidUpdate(prevProps, prevState, snapshot)：在更新后立即调用。
  3. 卸载阶段（Unmounting）：
    1. componentWillUnmount()：在组件卸载前调用，用于清理资源，如取消网络请求、移除监听器等。
  4. 错误处理阶段，包括 componentDidCatch(error, info) 方法，用于捕获子组件树中的错误。

. React 怎么优化渲染性能？

  1. 使用 React.memo
    对于函数组件，使用 React.memo 可以防止在 props 没有变化时重新渲染组件。React.memo 会对 props 进行浅比较，如果没有变化就不会触发重新渲染。
  1. 使用 shouldComponentUpdate 或 PureComponent
    在类组件中，shouldComponentUpdate 方法可以用于手动控制组件是否需要更新。PureComponent 通过浅比较 props 和 state 自动实现了这一功能。
  1. 优化渲染列表
    循环渲染使用 key 属性来帮助 React 识别哪些项被改变、添加或者删除，避免不必要的渲染。
  1. 使用懒加载组件
    使用 React.lazy 和 Suspense 来懒加载组件，可以减少初始加载时间。

  1. 减少渲染层级
    尽量减少组件树的层级，扁平化的组件结构可以减少渲染时间。

  2. 使用 useMemo 和 useCallback
    useMemo：用于缓存复杂计算的结果，避免在每次渲染时都重新计算。

  1. useCallback：用于缓存回调函数，避免函数在每次渲染时都重新创建。

. React 使用过哪些命令式方式使用组件？

  1. 使用 Refs 访问 DOM 元素
    Refs 允许你直接访问 DOM 元素或组件实例，可以在需要直接操作 DOM 的情况下使用。

  2. 使用 forceUpdate 方法:强制组件重新渲染

  3. 通过 forwardRef 和 useImperativeHandle 来实现自定义暴露给父组件的命令

. react 缓存怎么实现

  1. 使用 React 内部状态进行缓存
    在组件内部使用 useState 或 useReducer 进行简单的数据缓存。比如，当你从 API 获取数据后，可以将数据存储在组件的状态中，之后再次渲染时，如果数据已经存在，就不再重新请求。
  1. 使用 useMemo 和 useCallback 缓存计算结果和函数
    useMemo 和 useCallback 可以用来缓存函数的计算结果或缓存一个回调函数，这样可以避免不必要的重新计算或函数重新创建。
  1. 使用 LocalStorage 或 SessionStorage 进行持久化缓存
    可以将数据缓存到 localStorage 或 sessionStorage 中，这样即使用户刷新页面，数据依然存在。

. react 组件间通信方式

  1. Props（属性）
    父组件到子组件：
    父组件可以通过 props 将数据传递给子组件。

子组件到父组件：
子组件可以通过调用 props 中的回调函数将数据传递给父组件。

  1. Context API
    Context 提供了一个无需为每层组件手动添加 props，就能在组件树间进行数据传递的方法。
  1. Refs
    使用 ref 可以获取组件实例，从而调用实例的方法或访问其属性。
  1. Events（事件）
    通过在父组件中定义事件处理函数，并在子组件中触发该事件。
  1. Redux
    Redux 是一个独立于 React 的状态管理库，但它经常与 React 一起使用。

Action： 一个描述发生了什么的对象。 Reducer： 根据旧的 state 和 action 返回新的 state 的函数。 Store： 整个应用的状态中心。

  1. useReducer 和 useContext
    useReducer 和 useContext 可以一起使用，以实现类似 Redux 的状态管理，而不需要引入额外的库。
